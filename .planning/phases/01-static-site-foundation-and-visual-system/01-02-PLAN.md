# Plan 01-02: Shared Layout and Visual System

## Objective

Implement the reusable page shell and visual foundation locked by `01-UI-SPEC.md`: standalone gallery header, minimal footer, global CSS tokens, shared layout, and placeholder surfaces.

## Context

Read before execution:

- `.planning/phases/01-static-site-foundation-and-visual-system/01-CONTEXT.md`

- `.planning/REQUIREMENTS.md`
- `AGENTS.md`
- files created by Plan 01-01

## Threat Model

<threat_model>
**Scope:** Static UI shell and visual styling.

**Threats considered:**
- External asset dependency introduced by linking to Computingeducation bundles or images.
- Accessibility regression through invisible focus states or low contrast.
- Scope creep by adding migrated exhibit content in the foundation phase.

**Mitigations:**
- Recreate visual language with local CSS only.
- Implement semantic landmarks and visible focus styles.
- Keep page body placeholder-only.

**Blocking severity:** A remote runtime asset dependency or missing focus visibility is blocking for this phase.
</threat_model>
