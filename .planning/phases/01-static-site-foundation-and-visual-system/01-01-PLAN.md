# Plan 01-01: Static Astro Foundation

## Objective

Create the minimal Astro/Vite/TypeScript static-site foundation for Recode.Remix.Design. This plan owns the build scaffold and route skeleton only; it must not implement final migrated exhibition content.

## Context

Read before execution:

- `.planning/PROJECT.md`
- `.planning/REQUIREMENTS.md`
- `.planning/ROADMAP.md`
- `.planning/phases/01-static-site-foundation-and-visual-system/01-CONTEXT.md`
- `.planning/phases/01-static-site-foundation-and-visual-system/01-UI-SPEC.md`
- `.planning/research/STACK.md`
- `AGENTS.md`

## Threat Model

<threat_model>
**Scope:** Static scaffold only.

**Threats considered:**
- Remote runtime dependency introduced accidentally through scaffold templates.
- Build scripts relying on a server runtime or CMS.
- Deployment URL drift causing wrong canonical/site metadata.

**Mitigations:**
- Use Astro static output only.
- Keep scripts local and deterministic.
- Set `site` explicitly to `https://gallery.computingeducation.de`.

**Blocking severity:** No high-severity threats expected in this plan.
</threat_model>
