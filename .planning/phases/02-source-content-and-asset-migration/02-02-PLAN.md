# Plan 02-02: Import Content and Download Local Media

## Objective

Migrate the main exhibition page and all manifest exhibits from WordPress REST content into local JSON files, while downloading required source media and rewriting media references to local static asset paths.

## Tasks

<task>
<name>Add migration script</name>
<files>
- `package.json`
- `scripts/migrate-source-content.mjs`
</files>
<action>
Create `scripts/migrate-source-content.mjs` that:
- reads `src/data/exhibit-manifest.json`
- fetches the main page and each exhibit page through REST
- uses `cheerio` to parse `content.rendered`
- removes or ignores WordPress theme/plugin chrome if present
- finds required media references in `img src`, `a href`, `srcset`, and common WordPress media attributes
- downloads source media into `public/assets/recode-remix-design/`
- rewrites source media URLs in migrated HTML to `/assets/recode-remix-design/...`
- preserves non-media external links unchanged
- writes `src/data/exhibition.json`
- writes one JSON file per exhibit under `src/data/exhibits/`
</action>
<verify>
- `rg "migrate:content" package.json`
- `npm run migrate:content`
</verify>
</task>

<task>
<name>Generate local content files</name>
<files>
- `src/data/exhibition.json`
- `src/data/exhibits/*.json`
</files>
<action>
For each generated JSON file include:
- `title`
- `slug`
- `sourceUrl`
- `sourcePageId`
- `contentHtml`
- `media`
- `externalLinks`

The main exhibition JSON should include `title`, `sourceUrl`, `sourcePageId`, `contentHtml`, `media`, and `externalLinks`.
</action>
<verify>
- `test -f src/data/exhibition.json`
- `test "$(find src/data/exhibits -name '*.json' | wc -l | tr -d ' ')" = "17"`
- `rg '"contentHtml"' src/data/exhibition.json src/data/exhibits`
</verify>
</task>

<task>
<name>Download and rewrite media locally</name>
<files>
- `public/assets/recode-remix-design/**`
- `src/data/exhibition.json`
- `src/data/exhibits/*.json`
</files>
<action>
Download all required WordPress media referenced by the migrated content. Reuse filenames when possible, adding disambiguation only when needed. Record local paths in each JSON file's `media` array.
</action>
<verify>
- `find public/assets/recode-remix-design -type f | head`
- `! rg "experimente\\.joachim-wedekind\\.de/wp-content" src/data`
</verify>
</task>

## Acceptance Criteria

- The main exhibition content and all 17 exhibit contents exist locally.
- Required source media exists under `public/assets/recode-remix-design/`.
- Migrated content uses local media paths, not WordPress media runtime URLs.
- Non-media external links are preserved in `externalLinks`.
