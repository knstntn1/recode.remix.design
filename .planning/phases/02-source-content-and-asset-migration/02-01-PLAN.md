# Plan 02-01: Build Source Inventory and Exhibit Metadata Map

## Objective

Create a reproducible source inventory that records the main exhibition source page and all 17 exhibit pages with stable local slugs, source URLs, REST IDs, and ordering.

## Tasks

<task>
<name>Add parser dependency and inventory script</name>
<files>
- `package.json`
- `package-lock.json`
- `scripts/source-inventory.mjs`
</files>
<action>
Install `cheerio` for robust HTML parsing. Create `scripts/source-inventory.mjs` that:
- fetches REST page `590`
- parses exhibit links from the main page content
- resolves known exhibit source URLs to REST page IDs
- handles `500 Linien` explicitly as page `625`
- writes `src/data/exhibit-manifest.json`
- supports `--check` mode that verifies the committed/generated manifest without changing files
</action>
<verify>
- `rg "cheerio" package.json`
- `npm run inventory:source`
- `node scripts/source-inventory.mjs --check`
</verify>
</task>

<task>
<name>Create stable exhibit manifest</name>
<files>
- `src/data/exhibit-manifest.json`
</files>
<action>
Ensure the manifest contains exactly 17 exhibits with:
- `order`
- `title`
- `slug`
- `sourceUrl`
- `sourcePageId`
- `sourceSlug`

Use local slugs suitable for `/exponate/{slug}/`, including `500-linien` and `punkt-linie-flaeche`.
</action>
<verify>
- `node -e "const m=require('./src/data/exhibit-manifest.json'); if (m.exhibits.length !== 17) process.exit(1)"`
- `rg '"slug": "500-linien"' src/data/exhibit-manifest.json`
- `rg '"sourcePageId": 625' src/data/exhibit-manifest.json`
</verify>
</task>

## Acceptance Criteria

- `scripts/source-inventory.mjs` exists and can regenerate/check the manifest.
- `src/data/exhibit-manifest.json` contains the main source page metadata and all 17 exhibits.
- The manifest resolves `500 Linien` to page `625`.
- No exhibit is missing or duplicated.
