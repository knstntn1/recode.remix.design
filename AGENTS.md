## Project

**Recode.Remix.Design**

Recode.Remix.Design is a simple static exhibition website for the art exhibition of the same name. It presents the already prepared German-language exhibition material from Joachim Wedekind's "Experimente" site in a new standalone web presence that visually fits into the look of computingeducation.de.

The site will have a start page for the exhibition and separate detail pages for each exhibit. It will later be embedded or hosted under `gallery.computingeducation.de`, so all required content and media should be available locally instead of depending on the original WordPress site.

**Core Value:** Visitors can browse the Recode.Remix.Design exhibition independently under the Computingeducation visual identity, with all exhibit information and media available without relying on the source site.

### Constraints

- **Deployment**: The site will later live under `gallery.computingeducation.de` - paths, assets, and links should be static-hosting friendly.
- **Independence**: All required images and media from the source pages must be copied locally - the new site must not depend on the original WordPress media URLs.
- **Visual identity**: Typography, color scheme, footer, and general page feel should align with computingeducation.de.
- **Content fidelity**: Existing prepared content should be transferred directly, with only necessary structural or technical cleanup.
- **Scope**: Keep this a simple static site, not a CMS or dynamic application.

## Technology Stack

## Recommended Stack
### Core Technologies
| Technology | Version | Purpose | Why Recommended |
|------------|---------|---------|-----------------|
| Astro | 6.3.1 | Static site generation, layouts, content pages | Fits content-heavy static sites with shared layouts and no runtime CMS requirement. Official docs list 6.3.1 as current. |
| Vite | 8.0.12 | Development server and production build tooling | Astro builds on Vite patterns; Vite is fast, static-hosting friendly, and supports multi-page/static assets well. |
| TypeScript | 6.0.3 | Type-safe content metadata and build scripts | Helpful for exhibit metadata, route lists, and content import validation without adding runtime complexity. |
| Node.js | 20.19+ or 22.12+ | Local build runtime | Vite's current docs require these versions. |
### Supporting Libraries
| Library | Version | Purpose | When to Use |
|---------|---------|---------|-------------|
| Playwright | 1.60.0 | Visual and navigation verification | Use for desktop/mobile screenshots and link checks after implementation. |
| sharp | latest | Image optimization/conversion | Use if migrated images need resizing, thumbnails, or format normalization. |
### Development Tools
| Tool | Purpose | Notes |
|------|---------|-------|
| npm scripts | Standard dev/build/preview entry points | Keep commands simple: `dev`, `build`, `preview`, `test`. |
| Static asset crawler/import script | Download source WordPress images/media locally | Should preserve attribution context and rewrite links to local paths. |

## Installation
## Alternatives Considered
| Recommended | Alternative | When to Use Alternative |
|-------------|-------------|-------------------------|
| Astro | Plain HTML/CSS/JS | Use plain HTML only if no shared layouts, no content metadata, and no build pipeline are desired. Here, 18 pages and local asset rewriting justify a light SSG. |
| Astro | Eleventy | Eleventy is also valid for static content, but Astro gives a more modern component/layout model and stronger built-in asset/story structure. |
| CSS modules/global CSS | Tailwind | Tailwind is unnecessary unless the existing project already uses it. The target is a specific Computingeducation-like visual system, not rapid utility-driven design. |
## What NOT to Use
| Avoid | Why | Use Instead |
|-------|-----|-------------|
| WordPress clone/CMS | The requested deliverable is a simple static site and should be independent from the source site. | Astro static build |
| Remote WordPress media URLs | Violates the independence requirement and can break if the source site changes. | Local `public/assets/...` copies |
| Heavy client-side framework | The site is informational; runtime JS should be minimal. | Server-rendered/static Astro pages |
| Exact Computingeducation source copy | The goal is visual fit, not copying a Gatsby app wholesale. | Reimplement the relevant visual language in local CSS |
## Stack Patterns by Variant
- Use normal root-relative routes (`/`, `/exponate/schotter/`).
- Keep `site` configured to `https://gallery.computingeducation.de` for canonical metadata.
- Keep asset URLs generated by Astro and avoid hard-coded absolute hostnames.
- Centralize base/site configuration.
